# Java Micro-SaaS Starter PRD（产品需求文档）

# 产品名称：Micro-SaaS Starter

## 版本：v1.0

## 目标用户：独立开发者 / 小团队 / 技术创业者

# 一、产品背景

## 1.1 背景问题

独立开发者在做 SaaS 时，70% 时间浪费在重复建设：

- 登录注册

- 权限系统

- 支付订阅

- API Key

- 管理后台

- AI 接入

- 邮件通知

这些功能 几乎每个 SaaS 都要重做一次，最终导致：

- 上线慢

- 试错慢

- 很难持续做产品

## 1.2 产品目标

打造一套可复用的 Java Micro-SaaS 引擎，帮助开发者：在 1–2 周内上线一个可收费 SaaS 产品

## 核心价值

|目标|描述|
|---|---|
|快速启动|提供完整 SaaS 基础设施|
|内置变现|默认支持订阅收费|
|AI Ready|内置 AI 能力|
|可复用|支持持续开发多个产品|

# 四、产品核心模块

系统分为 9 大模块（按开发阶段排序）

## 模块 1：认证与用户系统（Auth）

### 目标

提供完整用户身份体系

### 功能需求

#### 用户注册登录

- 邮箱注册

- 密码登录

- JWT 鉴权

- Refresh Token

#### 用户管理

- 修改资料

- 修改密码

- 登录日志

## 模块 2：Tenant（伪多租户）

### 目标

让系统未来可升级为团队 SaaS

### 功能需求

#### 初始策略

- 每个用户自动创建 workspace

- 数据全部属于 tenant

#### 数据规则

所有核心表必须包含：tenant_id

## 模块 3：RBAC 权限系统

### 目标

支持后台与 API 权限控制

### 功能需求

#### 权限模型

User → Role → Permission → API

#### 功能

- 创建角色

- 分配权限

- API 权限校验

## 模块 4：Billing 订阅系统（核心盈利模块）

### 目标

让任何产品可以直接收费 💰

### 功能需求

#### 套餐系统

- Free / Pro / Team

- 月付 / 年付

- 试用期

#### 订阅管理

- 创建订阅

- 升级套餐

- 降级套餐

- 取消订阅

#### 支付集成

- Stripe Webhook

- 订单记录

- 发票记录

## 模块 5：API Key + 用量计费

### 目标

支持 API / AI 类 SaaS

### 功能需求

#### API Key 管理

- 创建 API Key

- 禁用 / 删除

- 限流

#### 用量统计

- 每日调用量

- 套餐额度

- 超额限制

## 模块 6：AI Gateway（AI 中台）

### 目标

统一 AI 接入能力

### 功能需求

#### AI Provider 抽象

统一接口：

- /ai/chat

- /ai/embedding

- /ai/image

#### 使用统计

- Token 使用量

- 成本统计

- 租户维度统计

## 模块 7：Admin 后台

### 目标

支持系统运营

### 功能需求

#### 用户管理

- 查看用户

- 禁用用户

#### 订阅管理

- 查看订阅

- 手动调整套餐

#### 用量统计

- AI 使用量

- API 调用量

## 模块 8：异步任务系统

### 目标

支持生产环境

### 功能需求

- 邮件发送

- AI 长任务

- 报表生成

## 模块 9：通知系统

### 功能需求

- 邮件通知

- Webhook 通知

- 站内通知
