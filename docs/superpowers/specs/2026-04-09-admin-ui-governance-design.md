# 管理端后台重构设计（admin_ui）

## 1. 目标

将原 `ui/` 目录中的原型能力升级为“治理中台型”管理后台信息架构，输出可直接用于产品评审与前端实现对照的原型目录 `admin_ui/`。

核心目标：

1. 页面结构从“功能平铺”升级为“治理域分组”
2. 强化租户治理、权限审计、计费财务、AI 运行治理与运营告警
3. 形成统一后台视觉与导航规范
4. 为后续 `src/views/admin/**` 代码实现提供页面蓝图

## 3. 视觉策略（基于 ui-ux-pro-max）

- 字体：Plus Jakarta Sans
- 色彩：专业深蓝灰体系（浅背景 + 强对比文本）
- 组件：高密度信息卡 + 表格 + 筛选条 + 操作抽屉（原型态）
- 交互：统一 hover/focus、操作 Toast 反馈、支持页面密度切换

## 4. 页面模板规范

每个后台页由以下模块构成：

1. Hero 标题区：说明页面职责
2. KPI 区：关键状态指标
3. 主内容区：表格/策略矩阵/任务列表
4. 侧边辅助区：治理建议、风险提示、快捷操作

## 5. 验收标准

1. `admin_ui/` 页面可独立打开并互相跳转
2. 导航分组清晰，当前页高亮正确
3. 页面覆盖治理中台核心模块
4. 原型交互按钮具备可见反馈
5. 编码为 UTF-8 无 BOM

## 6. 后续实现建议

1. 将 `admin_ui/pages/*.html` 映射为 Vue Admin 页面
2. 先实现 Dashboard、Tenant、RBAC、Billing、AI Gateway、Usage 六大优先页
3. 接入真实 API 后保留 IA，不改交互骨架
