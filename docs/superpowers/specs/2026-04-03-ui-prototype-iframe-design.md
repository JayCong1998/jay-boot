# 前端高保真原型设计（`ui` 目录，`index.html + iframe` 方案）

## 1. 背景与目标

基于《Java Micro-SaaS Starter 技术开发文档》，为管理后台 Web 端输出可评审、可演示的高保真原型。

目标：

- 每个子页面独立一个 HTML 文件
- 使用 `ui/index.html` 统一承载，通过 `iframe` 嵌入子页面
- 覆盖 MVP 核心闭环：认证、工作台、租户、RBAC、计费、API Key、用量、AI 网关
- 保持统一视觉语言，支持后续前端工程化落地参考

## 2. 范围与边界

### 2.1 本次包含

- 高保真静态原型（HTML/CSS/少量 JS）
- 页面切换交互（侧边导航切换 iframe）
- 全局视觉系统（颜色、字体、间距、卡片、按钮、表格样式）
- 10 个页面：2 个认证 + 8 个后台核心页面

### 2.2 本次不包含

- 真正后端接口联调
- Vue/TS 工程化改造
- 复杂图表库接入（使用高保真占位与静态示意）
- 真正登录态与权限逻辑

## 3. 信息架构与导航

### 3.1 主壳页面

- 文件：`ui/index.html`
- 结构：
  - 顶部栏：租户信息、搜索、通知、用户菜单
  - 左侧导航：业务模块入口
  - 主区域：iframe 容器
- 默认首页：`dashboard.html`

### 3.2 导航菜单

- 工作台（Dashboard）
- 租户管理
- 角色权限
- 订阅计费
- API Key
- 用量统计
- AI 网关
- 登录
- 注册

## 4. 文件结构设计

```text
ui/
├── index.html
├── assets/
│   ├── global.css
│   └── shell.js
└── pages/
    ├── auth-login.html
    ├── auth-register.html
    ├── dashboard.html
    ├── tenant.html
    ├── rbac.html
    ├── billing.html
    ├── apikey.html
    ├── usage.html
    └── ai-gateway.html
```

说明：

- `global.css` 负责全局设计变量与通用组件样式
- `shell.js` 负责 `index.html` 导航切换与 iframe `src` 控制
- 子页面尽量复用同一套视觉变量，避免样式割裂

## 5. 页面原型内容设计

### 5.1 `auth-login.html`

- 品牌区 + 登录表单
- 字段：邮箱、密码、记住我
- 操作：登录按钮、去注册链接、忘记密码提示（示意）

### 5.2 `auth-register.html`

- 注册表单
- 字段：邮箱、密码、确认密码、工作区名称
- 操作：注册按钮、协议提示、去登录链接

### 5.3 `dashboard.html`

- 核心指标卡（活跃用户、API 调用、AI 成本、错误率）
- 趋势图区域（高保真占位）
- 待办事项/风险提醒区

### 5.4 `tenant.html`

- 当前租户信息卡
- 基础设置区域（名称、计划、Owner）
- 成员列表 + 角色摘要

### 5.5 `rbac.html`

- 角色列表表格
- 权限矩阵区域
- 用户角色分配区
- 详情侧栏样式区（静态示意）

### 5.6 `billing.html`

- 套餐卡片（FREE/PRO/ENTERPRISE）
- 当前订阅状态卡
- 账单时间线与支付状态区
- 升级/续费操作入口

### 5.7 `apikey.html`

- API Key 列表（名称、权限范围、过期时间、状态）
- 新建 Key 表单/弹层样式示意
- 复制、禁用操作位

### 5.8 `usage.html`

- 用量指标卡（请求数、Token 数、成本）
- 日趋势图示意
- API 维度 Top 列表
- 配额进度条

### 5.9 `ai-gateway.html`

- Provider/Model 路由配置区
- 请求参数示例区
- 响应预览区
- 调用指标摘要卡

## 6. 视觉与交互规范

### 6.1 视觉风格

- 方向：现代企业级 SaaS 控制台
- 色彩：
  - 主色：Slate 深灰蓝
  - 强调色：Teal 青绿
  - 告警色：Amber 琥珀
- 字体：`Plus Jakarta Sans` + `Noto Sans SC`
- 背景：浅渐变 + 细纹理（非纯色平铺）

### 6.2 交互细节

- 首屏进入：容器淡入
- 指标卡：轻微错峰出现（stagger）
- 菜单切换：激活态高亮 + iframe 平滑过渡

## 7. 设计约束与一致性

- 页面只使用 GET/POST 场景表达（与项目 API 规范一致）
- 不使用测试数据硬编码在说明文本中，仅作业务示例值展示
- 页面层级不超过三层嵌套
- 保持可维护性：共享样式优先，避免每页重复 CSS

## 8. 验收标准

1. `ui/index.html` 可直接打开并完成导航切换
2. 侧边栏每个菜单都能切换到对应 iframe 子页面
3. `ui/pages` 下每个页面可独立打开查看
4. 视觉风格统一、页面信息完整、具备高保真观感
5. 无 TODO、无伪代码、无缺失资源引用导致的空白页

## 9. 风险与应对

- 风险：iframe 相对路径错误导致页面加载失败  
  应对：统一使用相对 `pages/*.html` 路径并逐页验证
- 风险：多页样式不一致  
  应对：抽取 `global.css` 统一变量与基础组件
- 风险：高保真实现时间较长  
  应对：先完成骨架与关键区块，再补充细节视觉
