# 用户端盈利闭环 UI 原型设计（product_ui）

## 1. 目标与范围

本次设计面向“个人创作者”用户，围绕 AI 文本结果交付，建立可盈利的订阅闭环原型。目标是把用户路径从获客到复购完整串联，优先验证转化效率与留存机制。

核心范围：

1. 官网拉新与价值传达
2. 登录注册与新手激活
3. 创作工作台与结果交付
4. 支付与订阅管理
5. 历史复用与邀请裂变
6. 帮助与异常兜底页面

## 2. 商业闭环设计

闭环路径定义为：

官网（价值认知） -> 注册登录（账号转化） -> 新手引导（首日激活） -> 工作台（核心使用） -> 结果交付（价值兑现） -> 支付页（订阅变现） -> 订阅中心（续费管理） -> 历史复用（留存） -> 邀请裂变（低成本增长）

变现策略：

- 主收费模型：订阅制（月付/年付）
- 转化策略：突出 Pro 档 + 年付省钱锚点
- 升级触发点：高级模板、导出能力、批量生成
- 留存策略：历史复写与连续创作场景
- 增长策略：邀请奖励额度

## 3. 信息架构

目录：`product_ui/`

- `index.html`：闭环总览
- `pages/home.html`：官网首页
- `pages/features.html`：功能价值页
- `pages/pricing.html`：订阅定价页
- `pages/auth-login.html`：登录页
- `pages/auth-register.html`：注册页
- `pages/onboarding.html`：新手引导
- `pages/workspace.html`：创作工作台
- `pages/result-delivery.html`：结果交付页
- `pages/history.html`：历史记录页
- `pages/checkout.html`：支付页
- `pages/payment-success.html`：支付成功页
- `pages/subscription.html`：订阅中心
- `pages/profile.html`：个人中心
- `pages/referral.html`：邀请裂变页
- `pages/help.html`：帮助中心
- `pages/not-found.html`：404 页面

## 4. 视觉与交互方向

结合 `ui-ux-pro-max` 检索结果，采用以下方向：

- Pattern：Pricing-Focused Landing
- Style：Vibrant & Block-based
- 主色：#EC4899，强调色：#F97316
- 字体：Plus Jakarta Sans + Manrope
- 氛围：高信息密度卡片 + 强 CTA + 轻动效反馈

交互规则：

- 所有可点击元素提供明显 hover/focus 状态
- 关键按钮使用转化导向文案
- 全局支持月付/年付视图切换（原型态）
- 原型操作统一通过 toast 反馈

## 5. 页面模块设计

### 5.1 获客与转化

- 官网：首页主价值、社证、闭环路径、主 CTA
- 功能页：模板能力、场景价值、升级触发说明
- 定价页：三档套餐、Most Popular、年付优惠

### 5.2 激活与交付

- 登录/注册：低阻力表单与权益引导
- 新手引导：三步配置，支持跳过
- 工作台：输入区 + 多版本结果区 + 升级触发
- 结果交付：复制/导出/复写，承接升级

### 5.3 留存与增长

- 历史页：检索、筛选、复写，强化复用
- 订阅中心：套餐状态、账单、续费控制
- 邀请裂变：邀请链接、奖励规则、记录
- 帮助中心：FAQ + 提工单
- 404：快速回流路径

## 6. 数据流与状态设计（原型层）

当前为静态原型，数据流以页面状态模拟为主：

- 订阅周期状态：`monthly/yearly`，本地存储记忆
- 操作反馈：按钮触发 toast 提示
- 导航状态：按 `data-page` 自动高亮
- 价格文本：通过 `data-price-monthly` / `data-price-yearly` 切换

## 7. 异常与兜底

- 提供 `not-found.html` 作为异常流量回收页
- 支付成功页提供回到工作台与订阅中心入口
- 帮助中心作为故障与疑问统一出口

## 8. 验收标准

1. 所有页面可独立打开并互相跳转
2. 导航高亮、面包屑、移动端标签可用
3. 订阅周期切换可联动价格文本
4. 关键按钮均有交互反馈
5. 目录结构清晰，可直接供前端开发对照实现

## 9. 后续实现建议

1. 将 `product_ui` 页面映射到 `jay-boot-frontend/src/views/user/**`
2. 接入真实 API 后替换原型动作逻辑
3. 优先埋点：注册转化、首日激活、支付转化、复写频次、续费率
